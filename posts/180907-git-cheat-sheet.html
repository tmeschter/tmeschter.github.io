Git Cheat Sheet

git

A bunch of commands I use all the time (at least, often enough) but can never remember.

Listing Tags

List all tags that start with "foo/":

git tag --list foo/*

Rebase

Rebase the current branch onto a target branch:

git rebase <target>

This finds where target and the current branch forked and rebases all the changes from that point to the current head onto target.

Rebase another branch onto a target branch:

git rebase <target> <topic>

Checks out topic and then rebases it as above.

Rebase a range of commits onto a target branch:

git rebase --onto <target> <base> <topic>

This takes all the commits after base up through topic and rebases them onto target.

Edit commits during the rebase:

git rebase -i ...

This lets you reorder, drop, squash, edit, etc. commits during the rebase.

Bisect

git bisect start
git bisect bad <bad_commit>
git bisect good <good_commit>

<follow the prompts>

git bisect reset

Managing Branches

I tend to end up with a lot of local branches and can't always remember which ones have already been merged--and can thus be deleted--and which ones contain in-progress work. The --merged and --no-merged options can help.

git branch --merged master

This shows the local branches that have already been merged into master--that is, the commits the branches point to can be reached from the commit pointed to by master. These will tend to be branches I can now delete with git branch -D.

git branch --no-merged master

On the other hand, this command shows the branches that haven't been merged into master. I find these need further review before I can decide what to do with them. Some may be in-progress work that I just had to put on the back burner for a while but intend to come back to; others may have been merged into different branches (like last-minute fixes to a release branch).
